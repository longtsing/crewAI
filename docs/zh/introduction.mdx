# 什么是 CrewAI？

**CrewAI 是一个精简、极速的 Python 框架，完全从零开始构建 — 完全独立于 LangChain 或其他代理框架。**

CrewAI 为开发者提供高级简单性和精确的低级控制，非常适合为任何场景创建自主 AI 代理：

- **[CrewAI 团队](/zh/guides/crews/first-crew)**：优化自主性和协作智能，让您能够创建 AI 团队，其中每个代理都有特定的角色、工具和目标。
- **[CrewAI 流程](/zh/guides/flows/first-flow)**：启用细粒度、事件驱动控制，单次 LLM 调用用于精确任务编排，并原生支持团队。

通过我们的社区课程认证的开发者超过 100,000 名，CrewAI 正在迅速成为企业级 AI 自动化的标准。

## 团队如何工作

<Note>
  就像公司有部门（销售、工程、营销）在领导下协同工作以实现业务目标一样，CrewAI 帮助您创建具有专业化角色的 AI 代理组织，协作完成复杂任务。
</Note>

<Frame caption="CrewAI 框架概览">
  <img src="/images/crews.png" alt="CrewAI 框架概览" />
</Frame>

| 组件 | 描述 | 主要功能 |
|:----------|:-----------:|:------------|
| **团队** | 顶级组织 | • 管理 AI 代理团队<br/>• 监督工作流<br/>• 确保协作<br/>• 交付成果 |
| **AI 代理** | 专业团队成员 | • 具有特定角色（研究员、作家）<br/>• 使用指定工具<br/>• 可以委派任务<br/>• 做出自主决策 |
| **流程** | 工作流管理系统 | • 定义协作模式<br/>• 控制任务分配<br/>• 管理交互<br/>• 确保高效执行 |
| **任务** | 个别分配 | • 具有明确目标<br/>• 使用特定工具<br/>• 融入更大的流程<br/>• 产生可操作的结果 |

### 一切如何协同工作

1. **团队**组织整体操作
2. **AI 代理**处理其专业化任务
3. **流程**确保顺畅协作
4. **任务**完成以实现目标

## 流程如何工作

<Note>
  虽然团队擅长自主协作，但流程提供结构化自动化，对工作流执行提供精细控制。流程确保任务可靠、安全、高效地执行，精确处理条件逻辑、循环和动态状态管理。流程与团队无缝集成，让您能够在高度自主性与精确控制之间取得平衡。
</Note>

<Frame caption="CrewAI 框架概览">

</Frame>

| 组件 | 描述 | 主要功能 |
|:----------|:-----------:|:------------|
| **流程** | 结构化工作流编排 | • 管理执行路径<br/>• 处理状态转换<br/>• 控制任务序列<br/>• 确保可靠执行 |
| **事件** | 工作流动作的触发器 | • 启动特定流程<br/>• 启用动态响应<br/>• 支持条件分支<br/>• 允许实时适应 |
| **状态** | 工作流执行上下文 | • 维护执行数据<br/>• 启用持久化<br/>• 支持可恢复性<br/>• 确保执行完整性 |
| **团队支持** | 增强工作流自动化 | • 在需要时注入自主性<br/>• 补充结构化工作流<br/>• 平衡自动化与智能<br/>• 启用自适应决策 |

## 何时使用团队 vs 流程

<Note>
  理解何时使用 [团队](/zh/guides/crews/first-crew) 与 [流程](/zh/guides/flows/first-flow) 是在您的应用程序中最大化 CrewAI 潜力的关键。
</Note>

| 使用场景 | 推荐方法 | 原因？ |
|:---------|:---------------------|:-----|
| **开放式研究** | [团队](/zh/guides/crews/first-crew) | 当任务需要创造性思维、探索和适应时 |
| **内容生成** | [团队](/zh/guides/crews/first-crew) | 用于协作创建文章、报告或营销材料 |
| **决策工作流** | [流程](/zh/guides/flows/first-flow) | 当您需要可预测、可审计的决策路径和精确控制时 |
| **API 编排** | [流程](/zh/guides/flows/first-flow) | 用于与多个外部服务按特定序列可靠集成 |
| **混合应用程序** | 组合方法 | 使用 [流程](/zh/guides/flows/first-flow) 编排整体流程，[团队](/zh/guides/crews/first-crew) 处理复杂子任务 |

### 决策框架

- **选择 [团队](/zh/guides/crews/first-crew) 当：** 您需要自主问题解决、创造性协作或探索性任务
- **选择 [流程](/zh/guides/flows/first-flow) 当：** 您需要确定性结果、可审计性或对执行的精确控制
- **两者结合当：** 您的应用程序需要结构化流程和自主智能区域

## 为什么选择 CrewAI？

- 🧠 **自主运行**：代理根据其角色和可用工具做出智能决策
- 📝 **自然交互**：代理像人类团队成员一样沟通和协作
- 🛠️ **可扩展设计**：轻松添加新工具、角色和功能
- 🚀 **生产就绪**：为真实应用程序的可靠性和可扩展性而构建
- 🔒 **安全优先**：考虑企业安全要求而设计
- 💰 **成本高效**：优化以最小化 token 使用和 API 调用
