## 概述

CrewAI 的 MCP DSL（领域特定语言）集成提供了**最简单的方式**将您的代理连接到 MCP（模型上下文协议）服务器。只需在您的代理中添加一个 `mcps` 字段，CrewAI 会自动处理所有复杂性。

<Info>
这是大多数 MCP 用例的**推荐方法**。对于需要手动连接管理的高级场景，请参阅 [MCPServerAdapter](/zh/mcp/overview#advanced-mcpserveradapter)。
</Info>

## 基本用法

使用 `mcps` 字段将 MCP 服务器添加到您的代理：

```python
from crewai import Agent

agent = Agent(
    role="研究助手",
    goal="协助研究和分析任务",
    backstory="具有高级研究工具访问权限的专家助手",
    mcps=[
        "https://mcp.exa.ai/mcp?api_key=your_key&profile=research"
    ]
)

# MCP 工具现在自动可用！
# 无需手动连接管理或工具配置
```

## 支持的引用格式

### 外部 MCP 远程服务器

```python
# 基本 HTTPS 服务器
"https://api.example.com/mcp"

# 带身份验证的服务器
"https://mcp.exa.ai/mcp?api_key=your_key&profile=your_profile"

# 带自定义路径的服务器
"https://services.company.com/api/v1/mcp"
```

### 特定工具选择

使用 `#` 语法从服务器中选择特定工具：

```python
# 仅从天气服务器获取预测工具
"https://weather.api.com/mcp#get_forecast"

# 仅从 Exa 获取搜索工具
"https://mcp.exa.ai/mcp?api_key=your_key#web_search_exa"
```

### CrewAI AOP 市场

从 CrewAI AOP 市场访问工具：

```python
# 包含所有工具的完整服务
"crewai-amp:financial-data"

# 从 AMP 服务获取特定工具
"crewai-amp:research-tools#pubmed_search"

# 多个 AMP 服务
mcps=[
    "crewai-amp:weather-insights",
    "crewai-amp:market-analysis",
    "crewai-amp:social-media-monitoring"
]
```

## 完整示例

以下是使用多个 MCP 服务器的完整示例：

```python
from crewai import Agent, Task, Crew, Process

# 创建具有多个 MCP 源的代理
multi_source_agent = Agent(
    role="多源研究分析师",
    goal="使用多个数据源进行综合研究",
    backstory="""具有网络搜索、天气数据、
    财务信息和学术研究工具访问权限的专家研究员""",
    mcps=[
        # 外部 MCP 服务器
        "https://mcp.exa.ai/mcp?api_key=your_exa_key&profile=research",
        "https://weather.api.com/mcp#get_current_conditions",

        # CrewAI AOP 市场
        "crewai-amp:financial-insights",
        "crewai-amp:academic-research#pubmed_search",
        "crewai-amp:market-intelligence#competitor_analysis"
    ]
)

# 创建综合研究任务
research_task = Task(
    description="""研究 AI 代理对业务生产力的影响。
    包括对远程工作的天气影响、与 AI 相关的
    财务市场趋势和关于 AI 代理框架的最新学术出版物。""",
    expected_output="""涵盖以下内容的综合报告：
    1. AI 代理业务影响分析
    2. 远程工作的天气考虑因素
    3. 与 AI 相关的财​​务市场趋势
    4. 学术研究引用和见解
    5. 竞争格局分析""",
    agent=multi_source_agent
)

# 创建并执行团队
research_crew = Crew(
    agents=[multi_source_agent],
    tasks=[research_task],
    process=Process.sequential,
    verbose=True
)

result = research_crew.kickoff()
print(f"研究完成，使用了 {len(multi_source_agent.mcps)} 个 MCP 数据源")
```

## 工具命名和组织

CrewAI 自动处理工具命名以防止冲突：

```python
# 原始 MCP 服务器有工具："search"、"analyze"
# CrewAI 创建工具："mcp_exa_ai_search"、"mcp_exa_ai_analyze"

agent = Agent(
    role="工具组织演示",
    goal="展示工具命名如何工作",
    backstory="演示自动工具组织",
    mcps=[
        "https://mcp.exa.ai/mcp?key",      # 工具：mcp_exa_ai_*
        "https://weather.service.com/mcp",         # 工具：weather_service_com_*
        "crewai-amp:financial-data"                # 工具：financial_data_*
    ]
)

# 每个服务器的工具根据服务器名称获得唯一前缀
# 这防止不同 MCP 服务器之间的命名冲突
```

## 错误处理和弹性

MCP DSL 设计为健壮和用户友好：

### 优雅的服务器故障

```python
agent = Agent(
    role="弹性研究员",
    goal="尽管服务器问题仍进行研究",
    backstory="适应可用工具的经验丰富研究员",
    mcps=[
        "https://primary-server.com/mcp",         # 主要数据源
        "https://backup-server.com/mcp",          # 主站故障时的备用
        "https://unreachable-server.com/mcp",     # 将被跳过并警告
        "crewai-amp:reliable-service"             # 可靠的 AMP 服务
    ]
)

# 代理将：
# 1. 成功连接到工作的服务器
# 2. 记录失败服务器的警告
# 3. 使用可用工具继续
# 4. 不会因服务器故障而崩溃或挂起
```

### 超时保护

所有 MCP 操作都有内置超时：

- **连接超时**：10 秒
- **工具执行超时**：30 秒
- **发现超时**：15 秒

```python
# 如果无响应，这些服务器将优雅超时
mcps=[
    "https://slow-server.com/mcp",        # 如果无响应，10 秒后超时
    "https://overloaded-api.com/mcp"      # 如果发现耗时 >15 秒将超时
]
```

## 性能功能

### 自动缓存

工具模式缓存 5 分钟以提高性能：

```python
# 第一次代理创建 - 从服务器发现工具
agent1 = Agent(role="第一个", goal="测试", backstory="测试",
               mcps=["https://api.example.com/mcp"])

# 第二次代理创建（5 分钟内）- 使用缓存的工具模式
agent2 = Agent(role="第二个", goal="测试", backstory="测试",
               mcps=["https://api.example.com/mcp"])  # 快得多！
```

### 按需连接

工具连接仅在工具实际使用时建立：

```python
# 代理创建快速 - 尚未建立 MCP 连接
agent = Agent(
    role="按需代理",
    goal="高效使用工具",
    backstory="仅在需要时连接的高效代理",
    mcps=["https://api.example.com/mcp"]
)

# MCP 连接仅在工具实际执行时建立
# 这最小化连接开销并提高启动性能
```

## 与现有功能的集成

MCP 工具与其他 CrewAI 功能无缝协作：

```python
from crewai.tools import BaseTool

class CustomTool(BaseTool):
    name: str = "custom_analysis"
    description: str = "自定义分析工具"

    def _run(self, **kwargs):
        return "自定义分析结果"

agent = Agent(
    role="全功能代理",
    goal="使用所有可用工具类型",
    backstory="具有全面工具访问权限的代理",

    # 所有工具类型一起工作
    tools=[CustomTool()],                          # 自定义工具
    apps=["gmail", "slack"],                       # 平台集成
    mcps=[                                         # MCP 服务器
        "https://mcp.exa.ai/mcp?key",
        "crewai-amp:research-tools"
    ],

    verbose=True,
    max_iter=15
)
```

## 最佳实践

### 1. 尽可能时使用特定工具

```python
# 好 - 仅获取您需要的工具
mcps=["https://weather.api.com/mcp#get_forecast"]

# 效率较低 - 从服务器获取所有工具
mcps=["https://weather.api.com/mcp"]
```

### 2. 安全处理身份验证

```python
import os

# 在环境变量中存储 API 密钥
exa_key = os.getenv("EXA_API_KEY")
exa_profile = os.getenv("EXA_PROFILE")

agent = Agent(
    role="安全代理",
    goal="安全使用 MCP 工具",
    backstory="安全意识代理",
    mcps=[f"https://mcp.exa.ai/mcp?api_key={exa_key}&profile={exa_profile}"]
)
```

### 3. 为服务器故障做计划

```python
# 始终包括备份选项
mcps=[
    "https://primary-api.com/mcp",       # 首选
    "https://backup-api.com/mcp",        # 备用选项
    "crewai-amp:reliable-service"        # AMP 后备
]
```

### 4. 使用描述性代理角色

```python
agent = Agent(
    role="天气增强市场分析师",
    goal="考虑天气影响分析市场",
    backstory="具有天气数据权限的财务分析师，用于农业市场见解",
    mcps=[
        "https://weather.service.com/mcp#get_forecast",
        "crewai-amp:financial-data#stock_analysis"
    ]
)
```

## 故障排除

### 常见问题

**未发现工具：**
```python
# 检查您的 MCP 服务器 URL 和身份验证
# 验证服务器正在运行且可访问
mcps=["https://mcp.example.com/mcp?api_key=valid_key"]
```

**连接超时：**
```python
# 服务器可能慢或过载
# CrewAI 将记录警告并继续使用其他服务器
# 检查服务器状态或尝试备用服务器
```

**身份验证失败：**
```python
# 验证 API 密钥和凭据
# 检查服务器文档了解所需参数
# 确保查询参数已正确 URL 编码
```

## 高级：MCPServerAdapter

对于需要手动连接管理的复杂场景，使用 `crewai-tools` 中的 `MCPServerAdapter` 类。使用 Python 上下文管理器（`with` 语句）是推荐方法，因为它自动处理启动和停止与 MCP 服务器的连接。
