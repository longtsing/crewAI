# CrewAI AOP API

欢迎使用 CrewAI AOP API 参考。此 API 允许您以编程方式与已部署的团队交互，实现与您的应用程序、工作流和服务的集成。

## 快速开始

<Steps>
  <Step title="获取您的 API 凭据">
    导航到 CrewAI AOP 仪表板中您的团队详细信息页面，并从状态选项卡复制您的 Bearer 令牌。
  </Step>

  <Step title="发现所需输入">
    使用 `GET /inputs` 端点查看您的团队期望的参数。
  </Step>

  <Step title="启动团队执行">
    使用您的输入调用 `POST /kickoff` 以启动团队执行并接收 `kickoff_id`。
  </Step>

  <Step title="监控进度">
    使用 `GET /status/{kickoff_id}` 检查执行状态并检索结果。
  </Step>
</Steps>

## 身份验证

所有 API 请求都需要使用 Bearer 令牌进行身份验证。在 `Authorization` 头中包含您的令牌：

```bash
curl -H "Authorization: Bearer YOUR_CREW_TOKEN" \
  https://your-crew-url.crewai.com/inputs
```

### 令牌类型

| 令牌类型 | 范围 | 用例 |
|:-----------|:--------|:----------|
| **Bearer Token** | 组织级访问 | 完整团队操作，适合服务器到服务器集成 |
| **User Bearer Token** | 用户范围访问 | 有限权限，适合用户特定操作 |

<Tip>
您可以在 CrewAI AOP 仪表板中团队详细信息页面的状态选项卡中找到两种令牌类型。
</Tip>

## 基础 URL

每个部署的团队都有自己唯一的 API 端点：

```
https://your-crew-name.crewai.com
```

将 `your-crew-name` 替换为您从仪表板获得的实际团队 URL。

## 典型工作流

1. **发现**: 调用 `GET /inputs` 以了解您的团队需要什么
2. **执行**: 通过 `POST /kickoff` 提交输入以开始处理
3. **监控**: 轮询 `GET /status/{kickoff_id}` 直到完成
4. **结果**: 从完成的响应中提取最终输出

## 错误处理

API 使用标准 HTTP 状态代码：

| 代码 | 含义 |
|------|:--------|
| `200` | 成功 |
| `400` | 错误请求 - 无效输入格式 |
| `401` | 未授权 - 无效 bearer 令牌 |
| `404` | 未找到 - 资源不存在 |
| `422` | 验证错误 - 缺少必需输入 |
| `500` | 服务器错误 - 联系支持 |
