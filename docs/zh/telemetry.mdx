## 遥测

<Note>
    默认情况下，我们不收集根据 GDPR 和其他隐私法规被视为个人信息的数据。
    我们确实收集工具名称和代理角色，因此建议不要在工具名称或代理角色中包含任何个人信息。
	因为没有收集个人信息，所以不必担心数据驻留问题。
	当启用 `share_crew` 时，会收集额外的数据，如果用户包含个人信息，这些数据可能包含个人信息。
    用户在启用此功能时应谨慎，以确保符合隐私法规。
</Note>

CrewAI 使用匿名遥测来收集使用统计数据，主要目标是增强库。
我们的重点是改进和开发用户最常用的功能、集成和工具。

理解这一点至关重要：默认情况下，**不收集个人数据**，涉及提示、任务描述、代理背景故事或目标、
工具使用、API 调用、响应、代理处理的任何数据或机密和环境变量。
当启用 `share_crew` 功能时，会收集详细数据，包括任务描述、代理背景故事或目标以及其他特定属性，
以提供更深入的见解。这种扩展的数据收集如果用户已将个人信息纳入其团队或任务中，可能包含个人信息。
用户在启用 `share_crew` 之前应仔细考虑其团队和任务的内容。
用户可以通过将环境变量 `CREWAI_DISABLE_TELEMETRY` 设置为 `true` 或将 `OTEL_SDK_DISABLED` 设置为 `true` 来禁用遥测（请注意，后者会全局禁用所有 OpenTelemetry 检测）。

### 示例：
```python
# 仅禁用 CrewAI 遥测
os.environ['CREWAI_DISABLE_TELEMETRY'] = 'true'

# 禁用所有 OpenTelemetry（包括 CrewAI）
os.environ['OTEL_SDK_DISABLED'] = 'true'
```

### 数据说明：
| 默认 | 数据                                      | 原因和具体内容                                                                                                       |
|:----------|:------------------------------------------|:----------------------------------------------------------------------------------------------------------------------------|
| 是       | CrewAI 和 Python 版本                 | 跟踪软件版本。例如：CrewAI v1.2.3, Python 3.8.10。无个人数据。 |
| 是       | 团队元数据 | 包括：随机生成的密钥和 ID、流程类型（如'sequential', 'parallel'）、内存使用布尔标志（true/false）、任务计数、代理计数。全部非个人信息。 |
| 是       | 代理数据 | 包括：随机生成的密钥和 ID、角色名称（不应包含个人信息）、布尔设置（verbose、启用委派、允许代码执行）、最大迭代次数、最大 RPM、最大重试限制、LLM 信息（见 LLM 属性）、工具名称列表（不应包含个人信息）。无个人数据。 |
| 是       | 任务元数据 | 包括：随机生成的密钥和 ID、布尔执行设置（async_execution、human_input）、相关代理的角色和密钥、工具名称列表。全部非个人信息。 |
| 是       | 工具使用统计 | 包括：工具名称（不应包含个人信息）、使用尝试次数（整数）、使用的 LLM 属性。无个人数据。 |
| 是       | 测试执行数据 | 包括：团队的随机生成密钥和 ID、迭代次数、使用的模型名称、质量分数（浮点数）、执行时间（秒）。全部非个人信息。 |
| 是       | 任务生命周期数据 | 包括：创建和执行开始/结束时间、团队和任务标识符。存储为带有时间戳的跨度。无个人数据。 |
| 是       | LLM 属性 | 包括：名称、model_name、模型、top_k、temperature 和 LLM 的类名。全部技术性、非个人数据。 |
| 是       | 使用 crewAI CLI 进行团队部署尝试 | 包括：正在部署的事实和团队 ID，以及是否尝试拉取日志，没有其他数据。 |
| 否        | 代理扩展数据 | 包括：目标描述、背景故事文本、i18n 提示文件标识符。用户应确保文本字段中不包含个人信息。 |
| 否        | 详细任务信息 | 包括：任务描述、预期输出描述、上下文引用。用户应确保这些字段中不包含个人信息。 |
| 否        | 环境信息 | 包括：平台、发布版、系统、版本和 CPU 计数。例如：'Windows 10', 'x86_64'。无个人数据。 |
| 否        | 团队和任务输入和输出 | 包括：输入参数和输出结果作为不可识别数据。用户应确保不包含个人信息。 |
| 否        | 全面团队执行数据 | 包括：团队操作的详细日志、所有代理和任务数据、最终输出。全部非个人数据和技术性。 |

<Note>
    "默认"列中的"否"表示此数据仅在将 `share_crew` 设置为 `true` 时收集。
</Note>

### 选择进一步遥测共享

用户可以通过在团队配置中启用 `share_crew` 属性为 `True` 来选择共享他们的完整遥测数据。
启用 `share_crew` 会收集详细的团队和任务执行数据，包括任务的 `goal`、`backstory`、`context` 和 `output`。
这能够更深入地了解使用模式。

<Warning>
    如果您启用 `share_crew`，如果个人信息已纳入团队配置、任务描述或输出中，收集的数据可能包含个人信息。
    在启用此功能之前，用户应仔细审查他们的数据并确保符合 GDPR 和其他适用的隐私法规。
</Warning>
